## Dictionary
1. **Algorithm** is “a sequence of computational steps that transform the *input* into the *output*” “in a finite amount of time” “for all problem instances”. It can be also described as “a tool for solving a well-specified *computational problem*” [@cormen_introduction_2022{pp. 5}].
2. **Computational problem** is a “problem that asks for a solution in terms of an *algorithm* <…> A computational problem can be viewed as a set of _instances_ or _cases_ together with a, possibly empty, set of _solutions_ for every instance/case. The question then is, whether there exists an algorithm that maps instances to solutions.” [@noauthor_computational_2024].
3. **Data structure** “is a way to store and organize data in order to facilitate access and modifications” [@cormen_introduction_2022{pp. 9}].
4. **Input** – some value or set of values, which algorithm takes [@cormen_introduction_2022{pp. 5}].
5. **Instance** or **case of a problem** generally “consists of the *input* (satisfying whatever constraints are imposed in the problem statement) needed to compute a solution to the problem. <…> Sometimes, when the problem context is known, problem instances are themselves simply called ‘problems’” [@cormen_introduction_2022{pp. 6}].
6. **Online algorithms** are “*algorithms* that receive their *input* over time, rather than having all the *input* present at the start” [@cormen_introduction_2022{pp. 11}].
7. **Output** – some value or set of values, which *algorithm* produces.
8. Sorting problem
9. Statement of the problem specifies in general terms the desired input/output relationship for problem instances, typically of arbitrarily large size.
